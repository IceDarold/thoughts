4. Теория: от соседей к латентному пространству

4.1. Повтор: что мы уже умеем

Кратко напоминаешь:

  • У нас есть матрица взаимодействий R размера U × I:
    • U — число пользователей,
    • I — число товаров (items),
    • R[u, i] = рейтинг / взаимодействие (0/1 или значение).
  • Мы умеем:
    • считать similarity между пользователями/товарами;
    • рекомендовать «по соседям» (user-based, item-based CF).

Проблемы:

  • матрица разреженная → мало пересечений;
  • сложно масштабировать на миллионы пользователей/товаров;
  • иногда CF даёт слишком «локальную» картинку и не выявляет общие паттерны.

4.2. Переход к latent space

Объяснение простым языком:

  • Представь, что у фильма/товара есть скрытые качества:
    • «насколько он романтичный»,
    • «насколько он драматичный»,
    • «насколько это sci-fi»,
    • «насколько это комедия»…
  • И у пользователя есть скрытые предпочтения:
    • «любит sci-fi»,
    • «не любит мелодрамы»,
    • «обожает боевики»…

Мы эти качества не видим напрямую, но можем их выучить так, чтобы:

  • если у пользователя и фильма похожие координаты в этом скрытом пространстве → высокий рейтинг/вероятность взаимодействия;
  • если они в «разных» регионах → низкий.

Формально:

  • каждому пользователю u сопоставляем вектор p_u ∈ ℝ^d,
  • каждому item’у i — вектор q_i ∈ ℝ^d,
  • d — размер latent space (например, 16/32/64).

Тогда:

[

\hat{r}{u,i} = \langle p_u, q_i \rangle = \sum{k=1}^d p_{u,k} \cdot q_{i,k}

]

Это и есть предсказанный рейтинг/скор.

4.3. Низкий ранг и приближение матрицы

  • Матрицу R можно считать как U × I табличку, но:
    • фактически поведение пользователей объясняется меньшим числом факторов.
  • Пример:
    • 1й фактор — «любовь к sci-fi»,
    • 2й фактор — «любовь к драмам»,
    • 3й — «любовь к комедиям»…
  • Если достаточно d=20 таких абстрактных факторов, то:
    • вместо хранения/обработки огромной R (миллионы × сотни тысяч),
    • мы храним P (U × d) и Q (I × d).

Идея:

[

R \approx P Q^T

]

Где:

  • P — матрица пользователей (U × d);
  • Q — матрица items (I × d);
  • PQ^T даёт нам аппроксимацию исходной R.

4.4. Функция потерь для explicit MF (без bias)

Пока берём простую версию: explicit рейтинги, нет bias’ов.

Есть множество наблюдаемых пар (u, i) (где r_ui известен).

Хотим выбрать P и Q, минимизируя:

[

L(P, Q) = \sum_{(u,i) \in \Omega} (r_{u,i} - \langle p_u, q_i \rangle)^2 + \lambda \left( \sum_u |p_u|^2 + \sum_i |q_i|^2 \right)

]

Где:

  • Ω — множество наблюдаемых interaction’ов (например, поставленных рейтингов);
  • первая часть — MSE на известных значениях;
  • вторая часть — L2-регуляризация (боремся с переобучением);
  • λ — коэффициент регуляризации.

Интуитивно:

  • если λ очень маленький:
    • можем перетюнить и идеально подогнать обучающие данные, но плохо обобщиться;
  • если λ очень большой:
    • модель будет слишком «плоской» и мало выражать различий.

4.5. Как оптимизировать: идея SGD

Не лезем глубоко в аналитику, даём интуицию:

  • У нас есть параметры:
    • все p_u (вектора пользователей),
    • все q_i (вектора items).
  • Можно обновлять их с помощью stochastic gradient descent:

    Для каждой обучающей пары (u, i):

    1. Считаем предсказание:

      [

      \hat{r}_{u,i} = \langle p_u, q_i \rangle

      ]

    1. Считаем ошибку:

      [

      e_{u,i} = r_{u,i} - \hat{r}_{u,i}

      ]

    1. Обновляем:
      p_u ← p_u + lr * (e_{u,i} * q_i - λ * p_u)
q_i ← q_i + lr * (e_{u,i} * p_u - λ * q_i)

      где lr — learning rate.

  • То есть:
    • если модель недооценила r_ui, мы сдвигаем p_u и q_i, чтобы scalar product стал больше;
    • если переоценила — наоборот.

4.6. Связь с user-/item-based CF

  • CF:
    • похожесть задаётся напрямую через R (observed interactions);
    • нужно хранить много similarity.
  • MF:
    • учится компактному представлению P и Q;
    • сходство пользователей/товаров вытекает из латентных векторов:
      • cosine между p_u и p_v,
      • cosine между q_i и q_j.

Плюс MF:

  • даёт возможность обобщать на пары (u, i), которых мы не видели:
    • если какие-то паттерны уже выучены в latent space.

5.2. Реализация простого MF (без bias)

Прямо на уроке пишем скелет:

import numpy as np

class SimpleMF:
    def __init__(self, n_users, n_items, n_factors=20, lr=0.01, reg=0.1, n_epochs=10):
        self.n_users = n_users
        self.n_items = n_items
        self.n_factors = n_factors
        self.lr = lr
        self.reg = reg
        self.n_epochs = n_epochs

        # Инициализация латентных факторов
        self.P = 0.01 * np.random.randn(n_users, n_factors)
        self.Q = 0.01 * np.random.randn(n_items, n_factors)

    def predict(self, u, i):
        return np.dot(self.P[u], self.Q[i])

    def fit(self, train_data, val_data=None):
        # train_data: numpy array shape (N, 3): [user_idx, item_idx, rating]
        for epoch in range(self.n_epochs):
            # перетасуем данные
            np.random.shuffle(train_data)

            for u, i, r in train_data:
                u = int(u); i = int(i)
                # предсказание
                r_hat = self.predict(u, i)
                e = r - r_hat

                # текущие вектора
                p_u = self.P[u]
                q_i = self.Q[i]

                # обновления
                self.P[u] += self.lr * (e * q_i - self.reg * p_u)
                self.Q[i] += self.lr * (e * p_u - self.reg * q_i)

            if val_data is not None:
                rmse = self.evaluate_rmse(val_data)
                print(f"Epoch {epoch+1}/{self.n_epochs}, val RMSE: {rmse:.4f}")

    def evaluate_rmse(self, data):
        se = 0.0
        for u, i, r in data:
            u = int(u); i = int(i)
            r_hat = self.predict(u, i)
            se += (r - r_hat) ** 2
        return np.sqrt(se / len(data))

На уроке:

  • объясняешь каждый шаг цикла:
    • зачем перемешивать данные,
    • что такое e = r - r_hat,
    • что делают регуляризационные термы.

Затем:

train_array = train_df[["user_idx", "item_idx", "rating"]].values
val_array = val_df[["user_idx", "item_idx", "rating"]].values

model = SimpleMF(n_users, n_items, n_factors=20, lr=0.01, reg=0.1, n_epochs=5)
model.fit(train_array, val_array)

Смотрим, как RMSE по эпохам падает (или не падает — тоже интересно, можно обсудить, что не так с lr/регуляризацией).

5.3. «Подглядывание» в latent space

  • Берём нескольких пользователей:
    u1, u2 = 0, 1
from numpy.linalg import norm

cos = np.dot(model.P[u1], model.P[u2]) / (norm(model.P[u1]) * norm(model.P[u2]) + 1e-8)
print("cosine similarity between user 0 and 1:", cos)
  • Смотрим:
    • каких items каждый из них любит (по исходным рейтингам),
    • насколько cosine-сходство между p_u1 и p_u2 соответствует их похожести.

Это даёт очень хорошую интуицию «пользователь как вектор».
