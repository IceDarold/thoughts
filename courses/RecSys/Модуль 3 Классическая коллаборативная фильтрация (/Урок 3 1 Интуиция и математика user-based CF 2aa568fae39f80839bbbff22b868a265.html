4. Интуиция user-based CF

4.1. История на пальцах

Пример: у нас есть четыре человека и фильмы.

  • Пользователь A: любит sci-fi и боевики.
  • Пользователь B: тоже sci-fi и боевики.
  • Пользователь C: обожает мелодрамы и комедии.
  • Пользователь D: только детские мультики.

Сюжет:

  • A и B оба высоко оценили фильмы Matrix, Inception, Interstellar.
  • A посмотрел ещё Mad Max, B — нет.
  • Вопрос: будет ли Mad Max интересен B?

Интуитивно: да, потому что A и B похожи, а A «одобрил» Mad Max.

Дальше формализуем.

4.2. Картина мира user-based CF

  • Каждый пользователь → вектор предпочтений по предметам.
  • Мы ищем k ближайших соседей пользователя u по этим векторам.
  • Чтобы предсказать, как u отнесётся к item i:
    • смотрим, как к нему относятся соседи;
    • усредняем их «отношение» с весами, зависящими от сходства.

5. Формализация: матрица R и обозначения

5.1. Матрица R

  • Пусть у нас:
    • пользователи: u ∈ U;
    • предметы: i ∈ I.
  • R[u, i] — «оценка» или «сила взаимодействия» пользователя u с item i:
    • в rating-задаче — рейтинг 1–5;
    • в implicit-задаче — например 0/1 (есть событие/нет).

Матрица разреженная: большинство R[u, i] = 0 или «отсутствует».

5.2. Обозначения

  • I(u) — множество items, с которыми взаимодействовал пользователь u.
  • U(i) — множество пользователей, которые взаимодействовали с item i.
  • sim(u, v) — мера сходства пользователей u и v.
  • N_k(u) — k ближайших соседей пользователя u (по sim).

В этом уроке можно считать, что:

  • мы сначала работаем с explicit рейтингами (для простоты);
  • потом покажем, что абсолютно та же идея работает с implicit.

6. Метрики сходства между пользователями

6.1. Косинусное сходство

Идея:

  • Представляем пользователя u как вектор R[u, :] в пространстве items.
  • Тогда косинусное сходство:

[

sim_{\cos}(u, v) = \frac{\sum_{i \in I(u) \cap I(v)} R[u, i] \cdot R[v, i]}{\sqrt{\sum_{i \in I(u) \cap I(v)} R[u, i]^2} \cdot \sqrt{\sum_{i \in I(u) \cap I(v)} R[v, i]^2}}

]

  • Считаем только по пересечению товаров, иначе всё обнуляется.
  • Интерпретация:
    • 1 — почти одинаковые предпочтения;
    • 0 — ортогональны, общих точек нет;
    • могут быть отрицательные значения, если рейтинги бывают отрицательные/отклонённые от среднего.

Важно: если пересечения почти нет, оценка «шумная».

6.2. Pearson корреляция (для рейтингов)

Когда есть рейтинги, полезно учитывать средний уровень строгости пользователя.

  • Многие люди ставят:
    • либо постоянно «5», если понравилось;
    • либо любят «3–4» и почти никогда не ставят «5».
  • Хотим сравнивать отклонения от среднего рейтинга.

Формула:

[

sim_{\text{Pearson}}(u, v) =

\frac{

\sum_{i \in I(u) \cap I(v)} (R[u, i] - \bar{R}_u) (R[v, i] - \bar{R}v)
}{
\sqrt{\sum{i \in I(u) \cap I(v)} (R[u, i] - \bar{R}u)^2}
\sqrt{\sum{i \in I(u) \cap I(v)} (R[v, i] - \bar{R}_v)^2}
}
]

где:

  • \bar{R}_u — средний рейтинг пользователя u;
  • \bar{R}_v — средний рейтинг пользователя v.

Интуиция:

  • Pearson говорит: «мы похожи, если нам оба фильм понравился/не понравился относительно наших личных средних».

6.3. Практические моменты

  • При маленьком количестве общих items:
    • sim ненадёжен → лучше добавлять минимальный порог по числу общих элементов.
  • Суперпользователи (у которых сотни/тысячи оценок):
    • могут быть похожи со всеми понемногу → разбавляют сходства.

7. Предсказание рейтинга/скора на основе соседей

7.1. Базовая формула (без сдвигов)

Простой вариант (особенно для implicit):

[

\hat{r}{u, i} =
\frac{
\sum\limits{v \in N_k(u),\ i \in I(v)} sim(u, v) \cdot R[v, i]
}{
\sum\limits_{v \in N_k(u),\ i \in I(v)} |sim(u, v)|
}
]

Интерпретация:

  • берём только соседей v, у которых есть оценка/взаимодействие с item i;
  • берём взвешенную сумму их рейтингов/сигналов;
  • нормируем на сумму модулей сходств.

7.2. Формула для рейтингов с учётом средних

Более реалистично для explicit:

[

\hat{r}_{u, i} =

\bar{R}u +
\frac{
\sum\limits{v \in N_k(u),\ i \in I(v)} sim(u, v) \cdot ( R[v, i] - \bar{R}v )
}{
\sum\limits{v \in N_k(u),\ i \in I(v)} |sim(u, v)|
}
]

Смысл:

  • сосед даёт вклад насколько ему этот фильм понравился/не понравился относительно его собственного среднего;
  • мы прибавляем этот вклад к среднему рейтингу пользователя u.

7.3. Выбор k и фильтрация соседей

  • k обычно от 20 до 100 для практических задач (зависит от sparsity).
  • Фильтрация:
    • игнорировать пользователей с малым числом пересечений;
    • игнорировать соседей с очень маленьким по модулю sim (например, |sim| < 0.1).

8. Построение топ-K рекомендаций из user-based CF

Порядок шагов:

  1. Для пользователя u:
    • найдём его N_k(u) — k ближайших соседей.
  1. Соберём множество кандидатов items:
    • C(u) = ∪_{v ∈ N_k(u)} I(v) \ I(u)

      (все items, которые оценивали соседи, но ещё не оценивал u).

  1. Для каждого i ∈ C(u) посчитаем \hat{r}_{u, i} по формуле выше.
  1. Отсортируем items по \hat{r}_{u, i} и выберем топ-K.
  1. (Опционально) применим фильтры:
    • цена;
    • категория;
    • доступность;
    • бизнес-правила.

9.2. Косинусное сходство между двумя пользователями

Простейший вариант:

def cosine_similarity(u_vec, v_vec):
    # u_vec, v_vec — numpy-вектора размера num_items
    mask = (u_vec != 0) & (v_vec != 0)
    if not np.any(mask):
        return 0.0

    u_masked = u_vec[mask]
    v_masked = v_vec[mask]

    num = np.dot(u_masked, v_masked)
    den = np.linalg.norm(u_masked) * np.linalg.norm(v_masked)
    if den == 0:
        return 0.0
    return num / den

На занятии можно обсудить:

  • что будет, если считать по всем items (с нулями);
  • как оптимизировать (разреженные представления).

9.3. Поиск k ближайших соседей пользователя

def topk_similar_users(user_idx, R, k=50):
    # R: матрица (num_users, num_items)
    u_vec = R[user_idx]
    sims = []
    for v_idx in range(R.shape[0]):
        if v_idx == user_idx:
            continue
        sim = cosine_similarity(u_vec, R[v_idx])
        sims.append((sim, v_idx))

    sims.sort(reverse=True, key=lambda x: x[0])
    return sims[:k]

(Сразу проговариваем, что это наивно и медленно, но для учёбы ок.)

9.4. Предсказание рейтинга

def predict_rating_user_based(user_idx, item_idx, R, k=50):
    neighbors = topk_similar_users(user_idx, R, k=k)

    num = 0.0
    den = 0.0
    for sim, v_idx in neighbors:
        r_vi = R[v_idx, item_idx]
        if r_vi == 0:
            continue
        num += sim * r_vi
        den += abs(sim)

    if den == 0:
        return 0.0  # или средний рейтинг по item / пользователю
    return num / den

9.5. Рекомендации для пользователя

def recommend_user_based(user_idx, R, k_neighbors=50, k_recs=10):
    neighbors = topk_similar_users(user_idx, R, k=k_neighbors)

    # items, с которыми взаимодействовал user
    user_items = set(np.where(R[user_idx] != 0)[0])

    candidate_scores = defaultdict(float)
    candidate_weights = defaultdict(float)

    for sim, v_idx in neighbors:
        if sim <= 0:
            continue
        v_items = np.where(R[v_idx] != 0)[0]
        for i_idx in v_items:
            if i_idx in user_items:
                continue
            candidate_scores[i_idx] += sim * R[v_idx, i_idx]
            candidate_weights[i_idx] += abs(sim)

    scores = []
    for i_idx in candidate_scores:
        if candidate_weights[i_idx] == 0:
            continue
        score = candidate_scores[i_idx] / candidate_weights[i_idx]
        scores.append((score, i_idx))

    scores.sort(reverse=True, key=lambda x: x[0])
    return scores[:k_recs]

10. Обсуждение ограничений user-based CF

Краткий блок обсуждения:

  • Sparsity:
    • у многих пользователей мало оценок → сложно найти «соседей»;
    • у новых пользователей (cold start) история почти пустая.
  • Сложность:
    • при большом числе пользователей sim(u, v) хранить и считать дороговато;
    • реальное решение: приближённые методы, отбор активных пользователей, переход к item-based CF.
  • Чувствительность к outliers:
    • странный пользователь с очень необычными оценками может влиять на рекомендации.

Завершение:

в следующих уроках мы посмотрим item-based CF (часто практичнее) и потом перейдём к implicit и более масштабным вещам.
