Урок 1.2. Популярностные рекомендации

1. Цели и результат урока

После урока студент:

  • Понимает разницу между:
    • глобальной популярностью;
    • популярностью в сегменте;
    • популярностью с учётом времени.
  • Умеет аккуратно считать:
    • pop(item) по разным критериям;

  • Умеет писать функцию рекомендаций:
    • с fallback’ами: «сначала сегмент, потом глобалка»;

  • Имеет в ноутбуке рабочий popularity-бейзлайн, который завтра уже можно засунуть в сабмит.

4. Блок 10–30 мин: теория — виды популярности

4.1. Global popularity

Идея:

pop(item) = сколько раз с item взаимодействовали (или сколько уникальных пользователей его трогали).

Обсуждаете:

  • Можно считать по:
    • всем событиям: view + click + cart + purchase;
    • только «сильным» событиям, например, покупкам;
  • Это уже ok-решение, особенно если:
    • у нас cold start по пользователю;
    • нет истории и фич.

Псевдоформулы:

  • pop(item) = count(interactions where item_id == item)
  • или pop(item) = #unique users, interacted with item.

4.2. Сегментная популярность

Идея:

популярность должна зависеть от сегмента пользователя.

Примеры сегментов:

  • гео: страна, город;
  • язык интерфейса;
  • возрастная группа;
  • тип устройства (mobile/desktop, Android/iOS);
  • простая кластеризация пользователей (потом).

Формально:

  • pop(item | segment = s) = количество взаимодействий с item среди пользователей сегмента s.

Обсуждение:

  • Если сегмент маленький → мало данных → шум.
  • Нужен fallback:
    • pop(item | segment)

      → если нет достаточной статистики → pop(item) (глобальная).

4.3. Time-decayed popularity

Проблема «вечных хитов»:

  • Товар мог быть суперпопулярным год назад, но сейчас его уже никто не трогает.
  • Старые события должны весить меньше, чем свежие.

Примеры decay:

  1. Окно по времени:
    • считаем популярность только за последние N дней:
      • pop_recent(item) = count(interactions for item in [T-N, T]).
  1. Экспоненциальный decay:
    • вес события:
      • w = exp(-λ * Δt),

        где Δt — сколько времени прошло с момента события.

    • pop(item) = Σ w_i по всем событиям item.
  1. Ступенчатый decay (piecewise):
    • события:
      • за последнюю неделю → вес 3;
      • за прошлый месяц → вес 2;
      • старше месяца → вес 1.

Обсудить простоту реализации vs качество.

На практике на первом этапе можно брать просто окно по времени.

4.4. Бизнес-правила и фильтры

Примеры:

  • Не рекомендуем:
    • товары, которые недоступны (out-of-stock);
    • товары, явно помеченные как запрещённые;
    • тот же item пользователю, если задача — показать новинки.
  • Простейшие правила:
    • ограничение на количество товаров из одной категории;
    • приоритет новинок.

Вывод блока: популярность ≠ один вектор. Это целое семейство базовых моделей.

5. Блок 30–65 мин: считаем популярности в ноутбуке

Всё ниже — как каркас, который можно давать студентам.

5.1. Global popularity

В ноутбуке:

import pandas as pd

train = pd.read_csv("../data/train_interactions.csv")

# допустим, у нас есть столбец event_type, и нас интересуют только 'purchase'
target_events = train[train['event_type'] == 'purchase'] if 'event_type' in train.columns else train

item_popularity = (
    target_events['item_id']
    .value_counts()
    .rename('popularity')
    .reset_index()
    .rename(columns={'index': 'item_id'})
)

item_popularity.head()

Дальше удобно иметь словарь item → score:

item_popularity_dict = dict(zip(item_popularity['item_id'], item_popularity['popularity']))
popular_items_global = item_popularity['item_id'].tolist()

5.2. Сегментная популярность

Выбираете один сегмент, который точно есть в данных, например country или user_group. Если его нет, можно сделать искусственный (например, user_id % 10 как «кластер»).

Предположим, что у нас есть колонка country в train:

# сначала присоединим user-атрибуты, если они в отдельном файле
# users = pd.read_csv("../data/users.csv")
# train = train.merge(users[['user_id', 'country']], on='user_id', how='left')

segment_col = 'country'

segment_item_popularity = (
    target_events
    .groupby([segment_col, 'item_id'])
    .size()
    .rename('popularity')
    .reset_index()
)
segment_item_popularity.head()

Теперь хотим быстро получать топ товары по сегменту. Например, сделаем словарь:

from collections import defaultdict

segment_to_items = defaultdict(list)

for seg, group in segment_item_popularity.groupby(segment_col):
    # сортируем по popularity для конкретного сегмента
    top_items = group.sort_values('popularity', ascending=False)['item_id'].tolist()
    segment_to_items[seg] = top_items

И fallback: если сегмента нет в словаре или мало данных, вернем popular_items_global.

5.3. Time-decayed popularity (упрощённый вариант)

Берём просто события за последнее окно, скажем 30 дней.

Допустим, timestamp — это unix time или что-то превращаемое в datetime:

train['timestamp'] = pd.to_datetime(train['timestamp'])

max_time = train['timestamp'].max()
window_days = 30
time_threshold = max_time - pd.Timedelta(days=window_days)

recent_events = target_events[target_events['timestamp'] >= time_threshold]

item_recent_popularity = (
    recent_events['item_id']
    .value_counts()
    .rename('recent_popularity')
    .reset_index()
    .rename(columns={'index': 'item_id'})
)

item_recent_popularity_dict = dict(zip(
    item_recent_popularity['item_id'],
    item_recent_popularity['recent_popularity']
))
recent_popular_items = item_recent_popularity['item_id'].tolist()

Можно показать второй вариант — ступенчатый decay:

def compute_time_weighted_popularity(df, now=None):
    if now is None:
        now = df['timestamp'].max()

    def weight_for_delta(delta_days):
        if delta_days <= 7:
            return 3.0
        elif delta_days <= 30:
            return 2.0
        else:
            return 1.0

    df = df.copy()
    df['delta_days'] = (now - df['timestamp']).dt.days
    df['weight'] = df['delta_days'].apply(weight_for_delta)

    weighted = (
        df.groupby('item_id')['weight']
        .sum()
        .rename('weighted_popularity')
        .reset_index()
    )
    return weighted

weighted_popularity_df = compute_time_weighted_popularity(target_events)
weighted_popularity_dict = dict(zip(
    weighted_popularity_df['item_id'],
    weighted_popularity_df['weighted_popularity']
))

Важно: не тонем в матане, просто показываем идею.

6. Блок 65–85 мин: пишем recommend_popularity(user_id, k)

6.1. Логика функции

Псевдо:

  1. Узнать сегмент пользователя (country, device…).
  1. Взять top-N из segment_to_items[segment], если сегмент есть и там достаточно товаров.
  1. При необходимости дополнить список глобальными популярными.
  1. Учитывать time-decayed score:
    • либо сортировать кандидатов по «смешанному» score:
      score = w_global * pop_global(item) + w_recent * pop_recent(item)
  1. Вернуть top-K.

6.2. Кодовый каркас

Предположим, что у нас есть:

  • user_features с колонками user_id и country;
  • заранее посчитанные:
    • item_popularity_dict,
    • item_recent_popularity_dict,
    • segment_to_items.
# подготовим mapping user_id -> segment (country)
user_segment = dict(zip(user_features['user_id'], user_features['country']))

def get_segment_for_user(user_id):
    return user_segment.get(user_id, None)

Теперь сама функция:

def recommend_popularity(user_id, k=10,
                         w_global=1.0, w_recent=1.0,
                         min_segment_candidates=30):
    """
    Простая популярностная рекомендация:
    - пытаемся использовать сегментные популярности
    - если их мало, используем глобальную
    - сортируем по комбинированному score (global + recent)
    """
    seg = get_segment_for_user(user_id)

    # 1. Берём кандидатов из сегмента, если есть
    candidates = []
    if seg is not None and seg in segment_to_items:
        candidates = segment_to_items[seg][:max(k * 5, min_segment_candidates)]

    # 2. Добавляем глобально популярные, чтобы расширить список
    candidates = list(dict.fromkeys(candidates + popular_items_global))  # убираем дубликаты

    # 3. Считаем score для каждого кандидата
    def score(item_id):
        g = item_popularity_dict.get(item_id, 0)
        r = item_recent_popularity_dict.get(item_id, 0)
        return w_global * g + w_recent * r

    ranked = sorted(candidates, key=score, reverse=True)

    # 4. Возвращаем top-K
    return ranked[:k]

Обсуждаете:

  • где тут можно «докрутить»:
    • ввести фильтры (не показывать уже купленное, если это условие задачи);
    • ограничить по категории и т.п.

Мини-практика

Пусть студенты:

  1. Выбирают 3–5 случайных пользователей.
  1. Для каждого:
    • вызывают recommend_popularity(user_id, k=10);
    • для сравнения — старый recommend_for_user_dummy.
  1. Пишут краткий комментарий:
    • есть ли видимая разница;
    • удалось ли поймать какой-то «сегментный» эффект.
