М11. Фичинжиниринг-системы

ниже — подробный Модуль 11. Фокус: системный фичинжиниринг: как за 5 часов построить воспроизводимую «фабрику признаков», которая даёт стабильный прирост, не ловит утечки и масштабируется на десятки миллионов строк.

1) Цели модуля

  • Спроектировать Feature Factory: конфиг-управляемую сборку признаков (groupby/статистики/кодировки/взаимодействия/время).
  • Встроить anti-leakage дисциплину: OOF-генерация, time-aware, контроль дрифта, «паспорт риска» для каждой фичи.
  • Настроить pipelines производительности: Polars/Parquet, кэш фич, инкрементальные пересчёты, контроль памяти/времени.
  • Поставить оценку фич: ablation, null-importance, permutation, SHAP-пакеты, «стоимость→прибавка».
  • Доставить «рёбра» в проект: features/registry.yaml, единый интерфейс fit_transform, юнит-тесты на утечки/детерминизм.

Выход: рабочая фабрика признаков с реестром, кэшем, тестами и отчётами по вкладу.

2) Формат и тайминг

  • 2 занятия × 2 часа (теория + лайв-код).
  • Самостоятельный мини-спринт 4–5 часов.
  • Защита: «паспорт фичсета» (1 стр) + таблица «ΔOOF vs стоимость».

3) Архитектура Feature Factory

Дерево проекта

src/features/
  ├── registry.yaml            # каталог фич и рецептов
  ├── base.py                  # интерфейсы: Feature, FitTransformFeature
  ├── groupby.py               # агрегаты по ключам
  ├── encoders.py              # freq/target/loo/hash/ordered (OOF-safe)
  ├── interactions.py          # кроссы, полиномы, биннинги, WoE
  ├── timeseries.py            # лаги/роллинги/decay (time-aware)
  ├── textlite.py              # лёгкие текстовые числовые фичи (tf/len/char)
  ├── selectors.py             # отбор по null/perm/mi/shap
  ├── cache.py                 # кэш+подписи (fingerprints), Parquet
  ├── validate.py              # анти-утечки, AV/PSI
  └── build_features.py        # оркестратор (CLI/typer), батчи, профилировка

Единый интерфейс

class Feature:
    name: str
    requires: list[str]   # зависимости (сырые поля/другие фичи)
    params: dict

    def fit(self, df, y=None, cv=None, time_key=None, groups=None): ...
    def transform(self, df): ...
  • Для OOF/time-aware фич — класс FitTransformFeature с методом fit_oof(...) → oof_values.
  • Registry загружает YAML и строит DAG фич (порядок, зависимости, повторное использование).

Конфиг-пример (registry.yaml)

features:
  - name: grp_user_item_stats
    kind: groupby
    params: {keys: [user_id,item_id], aggs: {price: [mean,std]}, min_count: 5}
    leakage_guard: oof
  - name: enc_target_item
    kind: encoder.target_kfold
    params: {key: item_id, smoothing: "m_estimate", noise: 0.01}
    leakage_guard: oof
  - name: ts_item_rolling
    kind: ts.rolling
    params: {entity: item_id, windows: [7,28], stats: [mean,std], lags: [1,7]}
    leakage_guard: time

4) Anti-leakage дисциплина (встроено в систему)

  • OOF-режим по умолчанию для таргет-зависимых фич. Храним fold_map.parquet, генерим OOF-столбцы и запрещаем fit на всём train.
  • Time-aware для TS: лаги/роллинги/expanding только до t, gap ≥ max_lag.
  • Group-изоляция: агрегаты/кодировки по ключам — внутри train-fold, группы не пересекаются.
  • Авто-чек: validate.py ловит: пересечения индексов, утечки по времени, использование теста, «слив» фичи в AV.

5) Кэш и подписи

  • Fingerprint набора данных: data_signature = sha1(schema + nrows + head_hash).
  • Fingerprint рецепта фичи: feature_signature = sha1(code_version + params + dependencies + fold_map).
  • Кладём Parquet в feature_store/{data_sig}/{feature_name}/{feature_sig}.parquet.
  • Инкрементальный пересчёт: если сигнатуры не менялись — берём из кэша; изменился один рецепт — пересчитываем только его и зависимых.

6) Производительность и память

  • I/O: Parquet + PyArrow, чтение Polars scan_parquet (ленивые вычисления), write zstd, row-group ~128MB.
  • Join: сортировка ключей, asof-join для time-aware, индексация категорий → Int32.
  • Groupby-рецепты:
    • Большие аггрегации → single pass: несколько стат в одном проходе.
    • Кроссы A×B → делать через hash/freq либо top-K + rare-bucket.
  • Батч-инференс: N строк chunksize, прогресс-бар + отчёт времени.
  • Профилировка: лог каждого шага (с) и потребление памяти (МБ).

7) Каталог «обязательных» рецептов (ядро)

A. GroupBy-агрегаты (safe)

  • count, nunique, mean, std, min, max, median, iqr, q01/q99 по ключам: user, item, merchant, geo, session, date-bucket.
  • Нормировки: per-entity z-score/robust-z, доли (x/sum_group), density (#events/день).
  • Recency: время от последнего события (safe для TS при time-aware).
  • Rare pooling: порог покрытия 99% → всё остальное в __RARE__.

B. Кодировки категорий (controlled)

  • freq/count (safe, OOF не нужен), target-kfold/LOO (OOF + сглаживание + шум), hash/binary для high-card.
  • CatBoost-ordered имитация: expanding mean per entity (TS-режим).

C. Взаимодействия и трансформы

  • Кроссы A×B → freq/hash; WoE/IV для скоринга; RankGauss/Yeo-Johnson для числовых с тяжёлыми хвостами.

D. Row-wise (анонимки)

  • row_sum/mean/std/skew/kurt, n_zero/pos/neg/miss, top-K суммы, кластерные агрегаты по корр-кластерам.

E. TimeSeries ядро

  • Лаги 1/7/28, окна [7,14,28,56] → mean/std/min/max, decay (EWMA α), сезонные дельты.

F. Text-lite (числовые)

  • Длины, токены/символы, tf-idf суммы по простому словарю, символьные шинглы (без сложных моделей).

8) Отбор и оценка фич

  • Ablation: greedy-forward (пачками), фиксированная модель/seed, лог: ΔOOF, time_cost, fit_cost.
  • Null-importance: перемешиваем таргет → распределение важности → отбрасываем «шумовые».
  • Permutation: по OOF, с повторениями; устойчивость = доля fold’ов с положительной прибавкой.
  • SHAP-пакеты: суммарная важность групп фич («агрегаты по user», «ts-ядро» и т.п.).
  • Matrix «прибавка/стоимость»: приоритезируем дешёвые и стабильные.

9) Паспорт фичи (шаблон, 6 строк)

  • name / рецепт / параметры
  • источник и ключи (entity/time)
  • anti-leakage: OOF/time/group, gap
  • стоимость: время, пик-память
  • прирост: ΔOOF ± std, устойчивость (fold-win %)
  • риск: AV-drift (↑/→/↓), комментарий

10) Диагностика дрифта и стабильности

  • AV (train vs valid) до/после пачки фич; PSI/KS по ключевым.
  • Сплит-стресс: другой seed/Group/Time → пересчитать ΔOOF; падение > X% → пометить «risky».
  • Seed-инвариантность: 5 сидов, разброс прироста.

11) Мини-практикумы

A) «Фич-ядро за 60 минут» (60–90 мин)

  1. Заполнить registry.yaml 10–15 safe-рецептами (groupby/freq/row-wise/ts).
  1. Собрать фичсет → кэш → бейзлайн → лог времени/памяти.
  1. Таблица «ΔOOF vs время».

B) «Отбор без самообмана» (60–90 мин)

  1. Null-importance → срез 30→15 фич.
  1. Permutation на 2 сплитах → финальные 10–12.
  1. SHAP-группы → отчёт.

C) «Дрифт-чек» (45–60 мин)

  1. AV/PSI до и после risk-пачки (target-encoding/TS-аггрегаты).
  1. Если AUC(AV) растёт — смягчаем/убираем; повторная оценка.

12) Самостоятельный 5-часовой спринт

Задание:

  • Для выбранной задачи (классификация/регрессия/TS) собрать фич-ядро (≥12 safe), добавить 1–2 «смелые» (target/TS), сделать отбор и выдать «паспорт фичсета».

Требования:

  • Корректный CV (из М1/М6), zero-leak дисциплина.
  • Логи: время/память на шаг, таблица «ΔOOF vs стоимость», AV/PSI отчёт.
  • 2 сабмита: базовый и улучшенный фичсет.

13) Чек-лист утечек (версия М11)

  • Любая статистика по таргету — только OOF; TS-агрегаты — только прошлое (gap).
  • Group-ключи не пересекаются между train/valid; fit не видит valid/test.
  • Реестр фич содержит поле leakage_guard; оркестратор фейлит сборку без него.
  • AV/PSI после каждой risk-пачки — обязательный отчёт.

14) Оценивание

  • Корректность (35%): отсутствие утечек, валидная CV/time политика.
  • Результат (30%): устойчивый ΔOOF/ΔLB, низкий std по фолдам, seed-инвариантность.
  • Инженерия (20%): кэш/подписи, скорость/память, чистые артефакты.
  • Отчёт (15%): «паспорт фичсета», матрица «прибавка/стоимость», решения по дрифту.

15) Что уходит «в рюкзак»

  • features/registry.yaml + шаблоны рецептов.
  • Модули: groupby.py, encoders.py, timeseries.py, interactions.py, selectors.py, validate.py, cache.py.
  • Оркестратор build_features.py (CLI: выбор профиля/режима OOF/сплита/батча).
  • Тесты: determinism, no-leak, schema-guard, time-guard.
  • Шпаргалки: «рецепты на 1 страницу», «OOF-кодировки», «TS-аггрегаты без утечки».

16) Встроено в 5-часовой формат (ритм)

  • 0–30 мин: выбрать профиль (классификация/регрессия/TS), заполнить registry.yaml safe-ядром, прогнать сборку → сабмит_1.
  • 30–90 мин: добавить пачку target/TS, OOF/time-guard, пересчёт → ΔOOF.
  • 90–150 мин: отбор (null→perm), финальный фичсет.
  • 150–210 мин: дрифт-чек (AV/PSI), стресс-сплит, паник-кнопка (урезать risky).
  • 210–240 мин: финальный сабмит, «паспорт фичсета», таблица «прибавка/стоимость».

Если хочешь, могу сразу накинуть скелет registry.yaml + каркасы классов Feature под твой текущий темплейт, чтобы команда завтра могла просто дописать рецепты и гонять спринты.
