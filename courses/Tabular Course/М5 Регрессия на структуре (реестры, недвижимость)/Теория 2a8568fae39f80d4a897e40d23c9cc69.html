Теория

ниже — «полная теория» для М5. Фокус: регрессия на структурированных данных с ярко выраженной пространственно-временной и иерархической природой (недвижимость, реестры, каталоги). Как строить устойчивые модели цен/стоимостей, не уехать в утечки, грамотно валидироваться, инженерить признаки (включая геопризнаки и соседей), выбирать лоссы и проверять переносимость.

1) Постановка и цели

Типовая задача: предсказать цену/стоимость (house price, appraisal), время/стоимость сделки, оценку объекта. Данные — структурированные: числовые и категориальные атрибуты объекта + гео (lat/lon, район), время (дата сделки), иерархии (квартира→дом→квартал→район→город), иногда — несколько таблиц (реестровые срезы, история продаж).

Типовые метрики: RMSE, MAE, RMSLE, MAPE/SMAPE; иногда кастомные стоимости ошибок.

Ключевые риски: утечки (время/агрегаты через валидацию), пространственный/временной сдвиг, зависимость наблюдений (соседние объекты), heavy-tail распределения, неоднородность по регионам, повторные продажи одного объекта.

2) Таргет и преобразования

  • Лог-таргет: (y'=\log(1+y)) — сглаживает хвосты, делает ошибки относительными → лучше при RMSLE/MAE-подобных целях и гетероскедастичности.
  • Цена за метр: (p_{m^2}= \frac{\text{price}}{\text{area_living}}) — нормирует по площади; далее предсказываем (p_{m^2}) и умножаем на площадь (контролируйте выбросы площадей).
  • Биннинг времени: месяц/квартал сделки для индексации тренда.
  • Стабильность метрики: если оф. метрика RMSE по «сырому» price, а вы обучаетесь на (\log price) — валидируйте в обеих шкалах (обратная трансформация с корректной биас-правкой).

Обратная трансформация лог-таргета:

Если модель даёт (\hat{z}=\log(1+y)), корректируйте смещение:

[

\widehat{y} \approx \exp(\hat{z} + \tfrac{1}{2},\widehat{\sigma}^2)-1

]

где (\widehat{\sigma}^2) — дисперсия ошибок в лог-пространстве (оцените по OOF).

3) Валидация: spatial/time/group — без этого всё мимо

  • GroupKFold по объекту/дому/кварталу: повторные продажи одного объекта не должны делиться между train/val.
  • Spatial CV: группируйте по геосетке (например, H3/геохеш на уровне района) — целые плитки уходят в валид; меняйте зерно сетки.
  • Time/Purged CV: если есть дата сделки, делайте роллинг с gap ≥ макс-лаг; обязательно оставьте OOT-период (последние месяцы).
  • Hybrid CV: Time × Spatial — валид по последним месяцам и на отдельных геоплитках (строже, но ближе к реальности).
  • Fold map фиксируем (fold_map.parquet) и переиспользуем в OOF-преобразованиях (см. ниже).

Симптомы плохой CV: большой shake-up CV↔LB, рост метрики при «странных» фичах (индексы, маски NaN), сильная зависимость от сидов.

4) Источники утечек в реестрах/недвижимости

  • Время: агрегаты «средняя цена в районе» считались с учётом будущих продаж; rolling без gap; признаки «время до следующей продажи».
  • Пространство: фичи соседей/района посчитаны на всём датасете (train+val) → валид «подсмотрен».
  • Дубликаты/near-dupes: одна и та же квартира (или почти идентичная) попала и в train, и в val.
  • Снапшоты реестра: признаки из состояний реестра после даты сделки (фактически look-ahead).
  • Текст/статусы: статус сделки/оценки, проставленный после события (частая скрытая утечка).

Простые анти-тесты: перестановка y внутри фолда (smoke test), AV (train vs test по X), PSI по ключевым фичам, поиск дубликатов (хэши, KNN по структурным полям).

5) Признаки: системная карта

5.1 Базовые числовые и производные

  • Площадь(и): общая/жилая/кухни, отношения (жилая/общая), плотность: площадь/комнату.
  • Комнаты/санузлы, удельные: санузлы/комнаты, окна/комнаты (если есть), этаж/этажность.
  • Возраст: ( \text{year_sale} - \text{year_built} ), факт капремонта (индикатор + возраст после ремонта).
  • Состояние/качество (если шкала): биннинг + монотонный ранг.
  • Лоты/дом: доля застройки (lot_area / building_area), этажность эффекты (верх/низ).
  • Выбросы: winsorize/clip по персентилам (например, [0.5; 99.5]) отдельно по районам.

5.2 Категориальные и иерархические

  • Зонирование, тип дома/материала, район, почтовый индекс.
  • Кодировки: freq/count (безрисково), OOF-TE (M-estimate, шум), иерархический back-off (квартира→дом→квартал→район→город).
  • Кроссы: район×тип дома (hash-cross/top-K), год×район (эффекты коинтеграции времени и локации).

5.3 Время

  • Месяц/квартал/год, сезонные индикаторы, праздники (если разрешено).
  • Трендовые индексы: (локальные) средние/медианы цен из прошлого (rolling/expanding с gap), только OOF.

5.4 Геопризнаки (безопасные)

  • Расстояния (Haversine) до CBD/центра/моря/реки/парков/школ (если доступны координаты POI).

    [

    d = 2R\arcsin\sqrt{\sin^2\frac{\Delta\varphi}{2} + \cos\varphi_1\cos\varphi_2\sin^2\frac{\Delta\lambda}{2} }

    ]

  • Плотности: количество объектов в радиусе r (фиксируйте r: 0.5/1/2 км).
  • Высота/уклон (если доступно) — часто коррелируют с престижностью/рисками.
  • Spatial binning: геосетка (геохеш/H3) как категория → freq/TE(OOF).
  • KNN-соседи (ООF): средняя/медианная цена соседей, считанная только по train-корпусу фолда (см. §6.2).

5.5 Фичи «истории продаж/реестров»

  • Время от последней продажи дома/квартиры, количество сделок за 6/12/24 мес (rolling из прошлого).
  • Стабильность продавца/застройщика: count/ratio по истории в регионе (group-agg по прошлому).
  • Версионность записи/состава: любые статусы — только на момент сделки.

5.6 Текст/мультимодальные (если можно)

  • Длина описания, количество «сильных» слов (ремонт, дизайнерский…), счётчики капслока/восклицаний — лёгкие прокси.
  • TF-IDF/эмбеддинги описаний — только OOF / fit на train-части фолда.

6) «Соседи», сглаживание и spatial leakage

6.1 Базовые соседские фичи

  • Local mean/median цены по ближайшим K объектам (или в радиусе r); K подбирайте кратко (5–20).
  • Kernel-сглаживание: веса (w_i \propto \exp(-d_i^2/2h^2)), ( \text{loc_price} = \frac{\sum w_i y_i}{\sum w_i}).

6.2 Только OOF!

Для каждого фолда k:

  1. Строим индекс соседей на train_k.
  1. Для val_k считаем локальные фичи по соседям из train_k (не заглядывая в val/test).
  1. Для test — по всему train (после обучения).

Запрет: считать локальные средние на train+val → утечка, CV завышен.

7) Лоссы и цели под регрессию цен

  • MAE / Huber — устойчивее к выбросам, ближе к «медианной» оценке стоимости.
  • RMSE / RMSLE — для симметричных ошибок в лог-шкале (цены часто лог-нормальны).
  • Quantile (pinball) — предсказание интервалов (P10/P50/P90) для оценщика.
  • Tweedie/Gamma — для положительных скью-распределений (если метрика допускает).

Практика: обучите базовую модель на (\log price) с Huber, доп. обучите quantile-LightGBM для интервалов → получите и точку, и доверие.

8) Модели и ограничения

  • GBDT (LGBM/XGB/Cat) — стандарт. Сдерживаем переобучение: глубина ≤6–8, min_data_in_leaf↑, lambda_l2↑, feature_fraction/bagging_fraction↓.
  • Монотонности (LGBM/XGB): для признаков с очевидной физикой (например, цена не убывает с ростом качественной площади)

    Используйте осторожно: для площади возможна конкавность (убывает удельная цена), поэтому лучше вводить площадь и 1/площадь/биннинг.

  • Стекинг L2: OOF-предикты LGBM/Cat/линейки → метамодель (линейная/ElasticNet).
  • Регуляризованные линейки (ElasticNet) — как интерпретируемый слой/компонент бленда.

9) Отбор и устойчивость признаков

  • Пермутационная важность по фолдам: избегаем «звёзд» на одном фолде и нуля на другом.
  • Stability selection: отбор на бутстрап-подвыборках, берём пересечение.
  • Leak-гварды: null-importance (важность на перемешанном y) — фичи с высокой null-важностью подозрительны (структурные).

10) Надёжность и переносимость

  • AV (train vs test): высокий AUC → spatial/temporal shift; проверьте геофичи/временные индексы.
  • PSI по ключевым фичам/скорингу: >0.25 — сильный сдвиг.
  • Alt-split: другой seed сетки/другая time-граница — прирост должен сохраняться.
  • Slice-оценки: районы/типы строений/квартали — где проваливаемся и почему.
  • Интервалы: quantile/конформал — сравнить покрытие 90/95% по срезам.

11) Реестры: мульти-табличность, связи, дедупликация

  • Ключи и связи: 1-к-N (дом→квартиры), M-к-N (объект↔сделка); держите snapshot на дату сделки.
  • Агрегации по иерархии: дом/квартал/район → mean/median/std цены в прошлом окне; count/ratio сделок; доли типов.
  • Entity resolution: дедупликаты по (адрес, площадь ±δ, этаж, год); блокировки по геосетке.

    Для KNN-поиска дублей используйте только train относительно val/test.

  • Версионность реестра: никакие пост-фактум статусы в признаки, только поля, доступные на (t_0).

12) Частые ошибки

  • Средние «цены по району» считали на всём датасете (или без time-gap) → лика.
  • Повторные продажи одного объекта разделили между train/val.
  • Геопризнаки соседей считали с участием валида/теста.
  • Таргет-энкодинги без OOF, эмбеддинги fit на train+val.
  • Оценка только на IID KFold → красивый CV, плохой LB на OOT/другом городе.

13) Мини-формулы и конструкции

Haversine (см. §5.4).

KNN-соседи (OOF):

[

\text{loc_y}(x) = \frac{\sum_{i \in \mathcal{N}_{\text{train}}(x)} w_i, y_i}{\sum w_i}, \quad w_i=\exp!\left(-\frac{d(x,i)^2}{2h^2}\right)

]

Quantile loss:

[

L_\tau(y,\hat{y}) = \max{\tau(y-\hat{y}), (\tau-1)(y-\hat{y})}

]

Huber:

[

L_\delta(r)=

\begin{cases}

\frac{1}{2}r^2,& |r|\le \delta\

\delta(|r|-\frac{1}{2}\delta),& |r|>\delta

\end{cases}

]

Иерархический back-off:

если (n_{\text{район}}<r) → (\lambda\cdot TE(\text{город})+(1-\lambda)\mu).

14) 5-часовой спринт (ритм)

0–30 мин — Диагностика

  • Зафиксировать CV: Spatial (геосетка) × Time (последние месяцы) × Group(объект).
  • Smoke-тест (shuffle-y), поиск дублей, AV/PSI грубо.

30–90 мин — Бейзлайн

  • Лёгкое ядро фич: лог-таргет, базовые числовые + freq/count по категориям, время (месяц/год), геосетка как категория.
  • LGBM/ Cat с консервативной регуляризацией. Проверка на OOT.

90–150 мин — Гео и история (без утечек)

  • OOF-KNN локальная цена, rolling индексы по прошлому (район-месяц).
  • Иерархический back-off для редких категорий. Alt-split/seed-bagging.

150–210 мин — Устойчивость

  • Пермутационная важность по фолдам, slice-оценки (районы/типы).
  • Quantile-модель (P10/P50/P90) для интервалов.

210–240 мин — Финал

  • Safe-бленд (2–3 модели/сида).
  • Короткий отчёт: CV-схема, анти-лики, гео-OOF детали, переносимость на OOT.

15) Чек-лист перед сабмитом

  • CV: Spatial×Time×Group зафиксирован; fold_map сохранён.
  • Все трансформы/агрегаты/соседи — OOF, без взгляда в валид.
  • Нет дублей train↔val; проверены near-dupes.
  • Геопризнаки и временные индексы рассчитаны из прошлого с gap.
  • Прирост подтверждён на alt-split; сид-стабильность ок.
  • Интервалы покрывают заявленный уровень; отчёт с AV/PSI и slice-таблицей.

16) Быстрые «рецепты», которые почти всегда работают

  • Лог-таргет + Huber, базовый LGBM, фичи: площадь/комнаты/возраст/этажность/качество, freq/count по районам/типам, месяц-сделки, геосетка-категория, OOF-локальная цена KNN.
  • Rolling-медиана цены по район×месяц из прошлого как сильный трендовый индикатор.
  • Иерархический back-off по гео: квартира→дом→квартал→район→город.
  • Quantile-добавка для доверия + бленд 2-3 сидов.

Если нужно, могу отдельно накинуть каркас OOF-KNN локальных цен (с безопасным индексом соседей и gap по времени) и шаблон Spatial×Time CV (на геосетке H3/геохеше) под твой пайплайн.
