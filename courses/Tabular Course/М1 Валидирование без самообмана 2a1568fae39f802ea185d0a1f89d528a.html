М1. Валидирование без самообмана

1) Цели модуля

  • Научиться распознавать структуру данных (IID / группы / время / ранжирование) и выбирать корректный сплит.
  • Собрать CV-фабрику (единый интерфейс для Stratified/Group/Time/Purged) + конфиги сплитов.
  • Освоить adversarial validation (AV) и базовые метрики дрифта (PSI/KS/Hellinger).
  • Наладить процесс контроля CV↔LB (корреляция, shake-up-риск, паник-план).

Выход модуля: «Паспорт валидации» для любого табличного кейса + готовые скрипты сплитов/AV.

2) Формат и тайминг

  • 2 занятия × 2 часа (теория + лайв-код).
  • 1 практикум-миниспринт (2–3 часа) между занятиями.
  • Защита «Паспорта валидации» (5–7 минут/чел).

3) Карта решений: как выбрать сплит за 15 минут

  1. Проверка структуры:
    • Есть ли время (timestamp, месяц, неделя, транзакции)?
    • Есть ли группы/идентификаторы (user_id, session_id, building_id, ticker)?
    • Задача IID или наблюдения зависимы (повторы, сессии, иерархии)?
  1. Выбор схемы:
    • IID классификация/регрессия → KFold / StratifiedKFold (k=5, repeats=1–3).
    • Группы (утечки внутри группы) → GroupKFold (k=5), иногда LeaveOneGroupOut.
    • Время → TimeSeriesSplit (+gap) или PurgedKFold (embargo).
    • Ранжирование/CTR с user/session → GroupKFold по сущности запроса/сессии.
    • Мультисущности во времени → rolling origin: сплиты по времени с удержанием групп.
  1. Параметры по умолчанию (разумные):
    • k=5, минимальный размер валидации ≥10% и ≥1k наблюдений (если доступно).
    • Для дисбаланса — stratify по таргету (классы/бинирование регрессии).
    • Для времени — gap (embargo) ≥ 1 шаг между train и valid.

4) Антипаттерны (поймать и обезвредить)

  • Target encoding на всём датасете, а не внутри фолда.
  • Лаги/роллинги с заглядыванием в будущее.
  • Разрезание групп между train/valid.
  • Подбор порога/метрик на публичном LB.
  • Выбор фич/гиперов «под приватный» через обратный инжиниринг публичного лидера (shake-up).

5) CV-фабрика (скелет)

Идея: единый конструктор сплитов + декларативные конфиги.

# config/cv.yaml
schema: time|group|iid|rank
folds: 5
repeats: 1
group_key: user_id
time_key: date
time_gap: 1D            # embargo/gap
stratify: target        # или bin(target, 10) для регрессии
seed: 42

API:

cv = CVFactory.from_config("config/cv.yaml").make_splitter(df)
for fold, (trn_idx, val_idx) in enumerate(cv.split(df, y, groups)):
    ...

Плюс: тест-набор «здравого смысла»:

  • нет пересечений индексов,
  • доли классов в каждом фолде ≈ исходной,
  • группы не «рвутся»,
  • при time/purged — нет «будущего» в train.

6) Adversarial validation (AV) за 10–15 минут

  • Слить train и test, метка is_test ∈ {0,1}.
  • Лёгкий бустинг на базовых фичах (без таргета!) → AUC(AV).
  • Интерпретация:
    • AUC(AV) ≤ 0.55 — сэмплы похожи; классический сплит ок.
    • 0.55–0.65 — умеренный дрифт; усиливаем регуляризацию/стабилизацию, подбираем сплит ближе к тесту.
    • ≥ 0.65 — сильный дрифт; нужен time/group-aware сплит, фильтрация токсичных фич, стресс-тест.
  • Используем AV-importance: выкидываем фичи, объясняющие «train vs test», если они искусственно подгоняют CV.

7) Метрики дрифта и стабильности

  • PSI по бинам/квантилям (числовые и закодированные категориальные).
  • KS-статистика, Hellinger — быстрая проверка распределений train vs test и train-fold vs valid-fold.
  • Стабильность OOF: std по фолдам; межфолдовая корреляция предсказаний; разброс метрик в RepeatedKFold.

8) CV ↔ LB: как держать связку

  • Вести таблицу сабмитов: sub_id, cv_score, lb_public, model_tag, features_tag, split_tag.
  • Смотреть корреляцию Spearman(cv_score, lb_public) по ≥3 сабмитам.
  • Если корреляция низкая:
    1. сменить сплит (Group/Time/Purged),
    1. смягчить фичи с высоким AV-importance,
    1. кросс-проверка на альтернативном hold-out.
  • План на shake-up: финальный ансамбль из 2–3 моделей с разными сплитами/фичсетами, веса по OOF.

9) Мини-практикум A (лайв-код, 60–90 мин)

Кейс: классификация с дисбалансом + группами (например, транзакции по пользователям).

Шаги:

  1. Бейзлайн с StratifiedKFold → OOF, сабмит_1.
  1. GroupKFold(user_id) → OOF, сабмит_2.
  1. AV (train vs test): AUC(AV), top-фичи AV → фильтрация/смягчение.
  1. Сравнить std OOF и CV↔LB, выбрать рабочую схему.

Артефакты: cv.yaml (2 схемы), отчет «почему выбран GroupKFold».

10) Мини-практикум B (лайв-код, 60–90 мин)

Кейс: временные данные (ежедневные продажи/счётчики).

Шаги:

  1. KFold (намеренно неверно) → показать переоценку CV.
  1. TimeSeriesSplit с gap → OOF, стабильность по фолдам.
  1. PurgedKFold с embargo → сравнить c TSS.
  1. PSI/KS между train-fold и valid-fold; выбор финального сплита.

Артефакты: cv.yaml (TSS vs Purged), графики распределений и дрейфа.

11) Инструменты и кодовые заготовки

  • Функция биннинга для регрессии: квантильные бины (m=10) → StratifiedKFold по бинам.
  • Генератор репитов: Repeated(Stratified)KFold для снижения дисперсии OOF.
  • Быстрый AV-скрипт: fit→AUC→feature_importance→droplist.
  • Стресс-тест сплита: перестановка сидов, перетасовка групп, проверка устойчивости.

12) Чек-лист «нет утечки» (версия модуля)

  • Кодировки по таргету — внутри фолда; хранить ключи фолдов.
  • Лаги/роллинги — только из прошлого; gap/embargo задан.
  • Группы целиком в train или valid; не делим.
  • Фичи, коррелирующие с is_test по AV, просматриваем вручную.

13) Мини-спринт (2–3 часа) между занятиями

Задача на выбор (преподаватель выдаёт датасет):

  • Вариант G (группы) или вариант T (время).

    Требования:

  • 2 альтернативные CV-схемы, AV-отчёт, PSI/KS для 5–10 топ-фич, 2 сабмита.
  • Короткий «Паспорт валидации» (1 стр): что за структура, какой сплит выбран и почему, риски, паник-план.

14) Оценивание

  • Корректность сплита (35%) — структура данных обоснована, утечек нет.
  • Стабильность (25%) — низкий разброс OOF, разумная корреляция CV↔LB.
  • Инструменты (25%) — рабочая CV-фабрика + AV/PSI/KS скрипты.
  • Отчёт (15%) — ясная аргументация и next steps.

15) Что берём в «рюкзак» после М1

  • cv.yaml + CVFactory (IID/Group/Time/Purged).
  • Скрипты AV и PSI/KS, шаблон отчёта «Паспорт валидации».
  • Набор sanity-тестов (shuffle-target, null-importance, repeated-CV).

16) Встроено в 5-часовой формат

  • Первые 20 минут спринта уходят на: выбор сплита → быстрый AV → фиксация решения в cv.yaml.
  • Дальше — только безопасные фичи, и никаких правок CV «по ощущениям» после первого сабмита.
