Модуль 12. Генеральная репетиция (6 часов, 2 задачи)

Модуль 12 — 6-часовая генеральная репетиция (2 задачи подряд). Цель — отработать весь боевой цикл под Всерос: быстрый старт, стабильный офлайн, валидный сабмит, паник-профиль, пост-мортем.

1) Формат и условия

  • Длительность: 6 ч.
  • Задач: 2 штуки, каждая ≈ 3 ч (разные домены).
  • Без LLM-генеративки (для неё отдельный курс).
  • Окружение: один репозиторий с заранее подготовленным каркасом (/projects/tabular, /recsys, /text_h3, /audio_kws, /cv_price).
  • Запрещено: менять сплиты под паблик, фитить трансформы на whole train, ковырять тест руками.
  • Что сдаём: 2 валидных сабмита + логи времени + короткий отчёт (пост-мортем).

2) Наборы задач (выбираем один набор в начале модуля)

Набор A (рекомендуемый):

  • Задача 1 — RecSys mAP@20 (Т-Шопинг).
  • Задача 2 — Табличка+гео MAE (Георейтинг).

Набор B (баланс текста/аудио):

  • Задача 1 — H@3 мульти-лейбл (Мыслим категориями).
  • Задача 2 — KWS harmonic (Не слышу тебя).

Набор C (тяжёлый):

  • Задача 1 — CV medianAPE (АвтоПрайс).
  • Задача 2 — RecSys mAP@20.

Выбирай набор под свой сильный трек; цель — одна “своя” + одна “соседняя”.

3) Разметка времени (ориентир)

На каждую задачу (3ч):

  • T+00:00–00:20 — распаковка данных, чек формата, запуск локальной метрики/сплита.
  • 00:20–01:10 — быстрый базелайн (готовые скрипты из модулей 2–7).
  • 01:10–02:10 — 1–2 апгрейда с наилучшим ROI (см. ниже по доменам).
  • 02:10–02:40 — пост-проц/калибровка под метрику, формат сабмита, sanity-тесты.
  • 02:40–03:00 — финализация, логирование, упаковка артефактов.

Перерыв между задачами 10 мин — переключение проекта, фиксация логов.

4) Артефакты, которые должны быть к концу

  • submit_task1.*, submit_task2.* — валидные файлы нужного формата.
  • logs/time_task*.json — тайминги этапов.
  • logs/metrics_task*.json — локальные метрики, параметры сплита.
  • release_check_task*.txt — результат скрипта проверки (формат/NaN/топ-k/20 items/ровно 3 метки и т.п.).
  • postmortem_task*.md — краткий отчёт: что сработало/не сработало, где узкие места.

5) Чек-листы «Definition of Done» (по доменам)

A) RecSys (mAP@20)

  • Hold-out = последние 7 дней, группы по user_id.
  • Кандидаты: co-vis + item2vec (+/- LightGCN) → union→dedup→budget (≤150–300).
  • Ранжер: LGBM/CatBoost (pointwise или lambdarank), фичи user/item/user×item/time.
  • Post-rank: dedup, fill-up до 20, tie-break, лёгкая диверсификация.
  • Сабмит: ровно 20 item_id на пользователя.

Быстрый ROI: добавить time_since_last_ui, co-vis max по последним L товарам, тренд item (неделя).

B) Табличка+гео (MAE → Score=1/(1+MAE))

  • Корректный KFold/Group/TimeSplit без утечек.
  • Гео-контрасты 300↔1000 м, соседства/плотности.
  • Отзывы → TF-IDF/эмбеддинги → агрегаты на id (только OOF).
  • Бленд Cat/LGBM, калибровка, клиппинг выбросов.
  • Сабмит: id,target в нужном диапазоне.

Быстрый ROI: добавить 10–20 контрастов _300m/_1000m, mean TF-IDF по отзывам, простой level-2.

C) Текст (H@3)

  • Итеративная стратификация, локальный hitrate@3.
  • TF-IDF + LR/SGD БЛ + эмбеддинговый ранжировщик или маленький BERT.
  • Пост-проц: всегда 3 метки, tie-break по priors.
  • Сабмит: .tsv в формате сэмпла.

Быстрый ROI: ранговый бленд LR + sentence-эмбеддинги, частотный бэк-офф для пустых описаний.

D) Аудио KWS (harmonic(1−FRR,1−FAR))

  • Декод Opus → mel, окна win/hop задано, кэш mel.
  • CNN/CRNN 3–5 блоков, class weights/SpecAugment.
  • Калибровка порога τ на вал-клипах (не accuracy).
  • Сабмит: id,label.

Быстрый ROI: сменить агрегатор клипа на top-k mean, тщательная калибровка τ.

E) CV цена (medianAPE)

  • TimeSplit (янв–ноя → train, декабрь → val-симуляция).
  • Фото-эмбеддинги (CLIP/ViT/ResNet) → агрегация 4→1 (mean/top-k/attention).
  • Табличка + фото → бленд/Level-2; Huber/Quantile, клиппинг.
  • Сабмит: ID,target.

Быстрый ROI: mean агрегация 4 фото + простой Level-2, квантильная регрессия.

6) Таймеры и «паник-профиль»

Триггеры паники (для каждой задачи):

  • T+40 мин — нет валидного сабмита: переключись на baseline-скрипт и добей формат.
  • T+90 мин — uplift офлайна < малой дельты vs базелайн: заморозь HPO, делай бленд/пост-проц.
  • T+120 мин — нет роста метрики: включи -fast профиль (урезанные фичи/кандидаты, без тяжёлых этапов), приоритезируй стабильный сабмит.

Паник-профили:

  • RecSys -fast: только co-vis + trending, ранжирование = сумма нормированных скорингов, без LightGCN.
  • Табличка -fast: только гео-контрасты, один бустинг, без текст-фич.
  • Текст -fast: LR на TF-IDF + жёсткий пост-проц «3 метки».
  • KWS -fast: предобученный CNN, агрегатор max, быстрая калибровка τ по валу.
  • CV -fast: только табличка или только mean эмбеддинги без Level-2.

8) Оценка и зачет

  • Проходной минимум: 2 валидных сабмита + офлайн-метрики рассчитаны корректно + пройден release_check.py.
  • Хорошо: ≥1 задачи с ощутимым uplifтом vs базелайн (по локальной метрике), лог времени ≤ заявленного бюджета.
  • Отлично: обе задачи с приростом; чёткий пост-мортем с причинами выигрыша/провала и планом улучшений.

9) Пост-мортем (шаблон на 1–2 страницы)

  • Контекст: какой набор задач, какое железо/время.
  • Что делал(а): шаги по минутам, что выкинул(а) и почему.
  • Метрики: базелайн → апгрейд 1 → апгрейд 2 → финал (таблица).
  • Риски/сдвиги: что ломало офлайн/паблик, как страховал(а).
  • Следующие 3 шага: что бы сделал(а) при +1 часу времени.
