Overview

Ниже — модульная программа ровно под типовые негенеративные форматы Всероса: табличка+текст/гео (MAE), мульти-лейбл текст (H@3), рекомендации (mAP@20), аудио KWS (harmonic mean 1−FRR/1−FAR), CV-регрессия по фото (medianAPE). Для каждого модуля: цели → темы → практика/контроль.

M0. Старт и боевой шаблон

Цели: за 30–40 минут поднимать репро-базелайн для любой из пяти задач.

Темы: структура репо; data/ src/ notebooks/; единый config.yaml; фиксация seed; CLI (train/predict/eval); локальный scorer; быстрые чек-листы сабмита.

Практика: завести каркас для пяти треков (tabular/text/recsys/audio/cv) с единым логгером.

M1. Метрики, CV и анти-утечки (сквозной модуль)

Цели: метрика и валидация совпадают с ЛБ, без сюрпризов.

Темы:

  • CV: KFold/Stratified/Group/TimeSplit + embargo; негласные «мины» (переток по id/дню/гео).
  • Метрики и их «чувствительность»: MAE (клиппинг/лог-трансф), H@3 (ровно 3 предсказания), mAP@20 (кандидаты + порядок), harmonic(1−FRR,1−FAR) (порог + баланс), medianAPE (медиана по MAPE, устойчивость к хвостам).

    Практика: реализовать 5 локальных scorer’ов, проверить на игрушечных примерах. Сверка с формулировками олимпиадных задач: MAE в «Георейтинге» ; H@3 в «Мыслим категориями» ; mAP@20 в «Т-Шопинг» ; harmonic в «Не слышу тебя» ; medianAPE в «АвтоПрайс» .

M2. Табличка + гео + текст (кейс «Георейтинг», MAE)

Цели: собрать сильный регрессор с текстовыми и гео-признаками.

Темы:

  • Гео-фичи: агрегации в радиусах (300/1000 м), плотности, контрасты 300↔1000, ближайшие POI.
  • Текст отзывов → признаки: TF-IDF/SG/эмбеддинги + агрегаты по заведению (mean/max, топ-n ключей).
  • Стабильный CV (Time/Group), защита от утечек.
  • Модели: CatBoost/LightGBM/XGB, target инженерия (лог-скейл), отбрасывание выбросов, калибровка.

    Практика: базелайн для «Георейтинга» с MAE-валидацией и сабмитом.

M3. Текст — мульти-лейбл классификация (H@3)

Цели: ровно три метки в правильном порядке, устойчиво к шуму.

Темы:

  • Бейзлайн: TF-IDF + логрег/SGDClassifier (One-Vs-Rest).
  • Дальше: sentence-эмбеддинги → cosine-rank top-k; простая переранжировка по priors/PMI.
  • Пост-процессинг: всегда топ-3, разруливание синонимов/редких классов, сглаживание частот.
  • CV по приложениям, balance классов, cut-offs.

    Практика: воспроизвести H@3 для «Мыслим категориями», сравнить два подхода (линейка vs эмбеддинги).

M4. RecSys I — candidate generation (mAP@20)

Цели: покрытие кандидатов ≥95%, за разумное время/память.

Темы:

  • Co-visitation матрицы (session/day/decay), временные окна и веса.
  • Item2vec/word2vec: окна, негативы, как получить 200–400 кандидатов.
  • LightGCN / простые графовые ходы (по времени хватит «лёгкой» реализации).
  • Объединение кандидатов, дедуп, business rules.

    Практика: на «Т-Шопинг» собрать candidate-gen и замерить recall@20.

M5. RecSys II — ranking + post-rank (mAP@20)

Цели: поднять позиционность и порядок.

Темы:

  • Фичи user×item×time: частоты, свежесть, co-vis скор, расстояния в эмбеддингах, позиционные признаки.
  • Ранкер (LGBM/CatBoost), групповой CV по пользователю; лист-wise vs point-wise.
  • Пост-процессинг: смешивание источников кандидатов, переранжировка, diversity/novelty, анти-повторы.

    Практика: полный конвейер на «Т-Шопинг» до устойчивого роста mAP@20.

M6. Аудио KWS (harmonic(1−FRR,1−FAR))

Цели: модель + правильный порог под метрику.

Темы:

  • Конвертация opus→wav, нормализация, мел-спектры (win/stride), specaugment.
  • CNN/AST-базелайн; валидация по спикеру/записи.
  • Тюнинг порога: grid → ROC-кривая в координатах (1−FRR, 1−FAR), выбор optimum; cost-sensitive loss.

    Практика: собрать KWS на «Не слышу тебя», вывести кривую, зафиксировать порог.

M7. CV-регрессия по фото (medianAPE)

Цели: смикшировать визуальные и табличные признаки.

Темы:

  • Фичи из изображений: CLIP/ViT эмбеддинги; агрегация до объявления (mean/max/attention-pool).
  • Табличные поля (мультивыбор → bag of options), лог-ценa, управление выбросами.
  • Стэкинг: Tabular + Image head; калибровка квантили/Huber.

    Практика: «АвтоПрайс»: baseline + два апгрейда, оценка по medianAPE, лог-пространство.

M8. Фичеринг и стэкинг (сквозной)

Цели: быстро растить очки без новых архитектур.

Темы: target/mean encoding c CV, frequency/rarity, временные лаги; level-2 бленд (ridge/Cat), баггинг/гладиентные бленды; out-of-fold протокол.

M9. HPO и бюджет времени

Цели: максимум очков за 1–2 часа тюнинга.

Темы: Optuna/Random Search, space по-умолчанию для Cat/LGBM; ранняя остановка; разумные лимиты.

M10. I/O и скорость

Цели: читать/писать большие паркет/аудио/изображения быстрее, чем соперники.

Темы: Polars/pyarrow, memory mapping, батчи, многопроцессность, кэш кандидатов для RecSys; контроль детерминизма.

M11. Диагностика, сдвиги и надёжность сабмита

Цели: не «ломать» сабмит на приватном тесте.

Темы: train↔test drift (частоты классов/товаров), стресс-тесты: пустые тексты, редкие категории, длинные хвосты цен; клиппинг/правила; валидация формата саба.

M12. Экзамен-симуляция (6 часов)

Цели: обкатка боевого распорядка.

Формат: 2×3 часа (RecSys/Tabular или Audio/CV) — с чистого листа до валидного сабмита; контроль списком «release checklist».

Результаты курса (что ученик умеет)

  • Табличка+гео+текст (MAE): воспроизводимый конвейер с анти-утечкой для «Георейтинга».
  • Текст H@3: мульти-лейбл, всегда ровно три метки, устойчивые пост-правила.
  • RecSys mAP@20: candidates→ranker→post-rank, без дублей, с быстрым I/O.
  • Аудио KWS: мел-пайплайн + грамотный порог под целевую метрику.
  • CV medianAPE: фьюжн фото+табличка, устойчивый к хвостам.
  • Единые утилиты: scorer’ы, CV-фабрика, блендинг, чек-листы и шаблоны сабмитов.
