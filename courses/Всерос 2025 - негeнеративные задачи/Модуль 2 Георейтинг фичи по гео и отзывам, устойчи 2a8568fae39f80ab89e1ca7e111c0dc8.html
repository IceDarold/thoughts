Модуль 2. Георейтинг: фичи по гео и отзывам, устойчивый MAE

Модуль 2 — «Табличка + гео + текст: Георейтинг (MAE)». Формат: сперва полная теория (что именно и почему даёт очки), затем пошаговая практика на реальном датасете All Cups «Георейтинг».

1) Результаты модуля (что ученик должен уметь)

  • Поднять надёжный CV без утечек и воспроизводить локальный MAE → Score = 1/(1+MAE) как на площадке.
  • Собрать сильные гео-признаки (радиусы 300м/1000м, плотности, контрасты 300↔1000, ближайшие POI/категории) из открытых полей задачи.
  • Извлечь признаки из отзывов (reviews), агрегировать их на уровне заведения, не допуская утечек.
  • Обучить и блендировать CatBoost/LightGBM/XGBoost, откалибровать предсказания в допустимый диапазон ответа (CSV id,target).
  • За 60–90 минут собрать чистый сабмит с проверенными форматами, который не развалится на приватном тесте.

2) Теория (структурно, 90–120 мин)

A. Постановка и данные

  • Задача: регрессия «комплексной оценки успешности» по данным о заведении, локации и отзывам. Сабмит — CSV(id, target).
  • Файлы: train.tsv.zip, test.tsv.zip, reviews.*.zip (отзывы к заведениям).
  • Гео/демо-признаки: уже даны аггрегаты в радиусах 300м и 1000м (постфиксы _300m, _1000m), базовые характеристики (traffic/homes/works/mean_income), демография (возраст, занятость, дети), группы дохода, интересы и т.д.
  • Текст отзывов: отдельный файл reviews (id, text) — привязка к заведению.
  • Метрика: MAE, на лидерборде промежуточный балл считают как Score = 1/(1+MAE).
  • Диапазоны таргета: в данных встречается описание «target 0–5 (0=нет рейтинга)», при этом формат ответа требует target как оценку 1–5 — учитываем это при калибровке/клиппинге.

B. Валидация и анти-утечки

  • CV: KFold/StratifiedKFold по бинам таргета; доп. вариант — GroupKFold по крупным гео-зонам/категориям (если есть признаки, создаём стабильные группы).
  • Анти-утечки:
    • агрегаты только по train-фолду, никакой информации из test;
    • текстовые признаки из reviews агрегируем на уровне id без использования какой-либо статистики теста;
    • избегать фичей, напрямую кодирующих id, и утечек через «знание будущего» (если строим окна по окрестности — делаем это на train-части).
  • Смысл CRISP-валидации: чтобы локальный MAE максимально коррелировал с LB-Score = 1/(1+MAE).

C. Гео-фичеринг (основа прироста)

  • Радиальные агрегаты: уже есть _300m/_1000m; строим контрасты: разности/отношения 300↔1000 (напр., density_300 / (density_1000+ε)), градиенты (локальность).
  • Плотность и «соседство»: число точек того же класса в радиусе; доля конкурентов в категории; «mix-index» разных категорий.
  • Ближайшие POI/центры масс: расстояние до ближайших популярных категорий, до «центра трафика».
  • Соц-демо: трансформы доходов, возрастных/семейных профилей, интересов (one-hot/freq/embeddings на группах признаков).
  • Нормализация по району: масштабирование признаков на локальную базу (z-score внутри радиуса 1000м).

D. Текст отзывов → признаки

  • Препроцесс: очистка, нормализация, токенизация; язык-агностичная лемматизация опционально.
  • TF-IDF/BoW, N-gram: fit по train-корпусу; агрегация на уровень id (mean/max TF-IDF, суммарные признаки, sentiment-proxy).
  • Sentence-эмбеддинги: усреднение эмбеддингов отзывов по заведению; отсечение экстремальных (robust-mean).
  • Контроль утечек: никаких fit на объединённом train+test тексте.

E. Моделирование и калибровка

  • Бейзлайны:
    • CatBoost (cat-фичи по входным категориям);
    • LightGBM/XGBoost с frequency/target encoding (с OOF-протоколом).
  • Блендинг: простые средние + level-2 (ridge/Cat) по OOF-предиктам.
  • Калибровка под ответ: таргет ограничиваем до [1,5] (формат саба), при этом учитываем, что в трейне 0 означает отсутствие рейтинга — полезна robust-регрессия или взвешивание низов, чтобы не «тянуть» в 0.
  • MAE-ориентированные приёмы: Huber/quantile-loss, лог-трансформация с обратным мэппингом, клиппинг выбросов предсказаний.

F. Диагностика/скорость/репрод

  • Быстрый I/O (parquet/polars), кэш промежуточных агрегатов, фикс seed.
  • Отчёт об абляциях: вклад гео-контрастов, вклад отзывов, вклад бленда.
  • Единый eval.py с MAE и расчётом Score.

3) Практика (лабы, 3–4 часа)

Лаба 1. Базовая сборка + корректный офлайн-скор

  • Распаковать train.tsv.zip, test.tsv.zip, подключить reviews. Проверить совпадение колонок, ID.
  • Реализовать cv.py (KFold с стратификацией по биннам таргета).
  • Написать metrics.py: mae() и score_mae = 1/(1+mae); прогнать на мини-сампле.
  • Бейзлайн: CatBoost/LightGBM на «сырых» гео-демо полях → локальный MAE.

Артефакт: baseline/submit.csv (id,target) в требуемом формате.

Лаба 2. Гео-контрасты и соседства

  • Построить блок фичей: контрасты _300m↔_1000m, доли конкурентов, «разреженность» района.
  • Включить нормализацию по району (делим на локальные медианы/стандарты).
  • Переобучить модель, снять прирост MAE; залогировать важности признаков.

Артефакт: features_geo.py и geo_report.md с топ-N фичей.

Лаба 3. Отзывы → табличные признаки без утечек

  • Сконструировать TF-IDF (uni/bi-gram), агрегировать на id: mean/max, доли тональностей/ключей.
  • Вариант 2: sentence-эмбеддинги (с усреднением по отзывам заведения, robust-trim).
  • Пересобрать модель, сравнить MAE; проверить, что fit делался только по train-fold.

Артефакт: features_text.py, проверка протокола OOF.

Лаба 4. Блендинг и калибровка предсказаний

  • Обучить CatBoost и LightGBM с разными наборами фичей; собрать OOF-предикты, сделать level-2 (ridge/Cat).
  • Откалибровать диапазон [1,5], проверить стабильность MAE при клиппинге и квантильной регрессии.
  • Снять финальный офлайн-MAE и посчитать Score.

Артефакт: blend.py, final_submit.csv.

Лаба 5. Надёжность и скорость

  • Стресс-тесты: редкие категории/пустые отзывы/аномальные районы; влияние на MAE.
  • Оптимизировать чтение/кэш/многопроцессность, замерить время полного пайплайна.
  • Чек-лист релиза: формат CSV, диапазон таргета, отсутствие NaN/inf, фикс seed.

4) Критерии сдачи модуля

  • Репозиторий с cv.py, metrics.py, features_geo.py, features_text.py, train.py, blend.py, eval.py, README.md (команды запуска).
  • Офлайн-MAE улучшен относительно бейзлайна минимум на X% (порог выставляем на курсе) и конвертирован в корректный Score по формуле.
  • Сабмит валиден (id,target), таргет в [1,5], проверен скриптом.
  • Протокол анти-утечек задокументирован (где и как исключали test-информацию, OOF-схема).

  1. Провести абляцию по группам фичей (контрасты 300↔1000, соседства, текстовые блоки).
  1. Сформировать отчёт: «Как изменился Score = 1/(1+MAE) после каждого блока? Какие признаки самые важные?».
